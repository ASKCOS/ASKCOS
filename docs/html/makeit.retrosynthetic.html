Keyword Arguments:
 
  • {number} -- Maximum number of reactions to allow before (max_depth) – stopping the recursive expansion down one branch (default: {3})
  • {number} -- Maximum number of precursor suggestions to (max_branching) – add to the tree at each expansion (default: {25})
  • {number} -- Time (expansion_time) – before searching the generated tree for buyable pathways (default: {240})
  • {number} -- Number of retrotransformer processes to fork for (nproc) – faster expansion (default: {1})
  • {number} -- Minimum number of precedents for an achiral template (mincount) – for inclusion in the template library. Only used when retrotransformers need to be initialized (default: {25})
  • {number} -- Minimum number of precedents for a chiral template (mincount_chiral) – for inclusion in the template library. Only used when retrotransformers need to be initialized. Chiral templates are necessarily more specific, so we generally use a lower threshold than achiral templates (default: {10})
  • {bool} -- Whether or not to pay close attention to chirality. When (chiral) – False, even achiral templates can lead to accidental inversion of chirality in non-reacting parts of the molecule. It is highly recommended to keep this as True (default: {True})
  • {string} -- Strategy used for template (template_prioritization) – prioritization, as a string. There are a limited number of available options - consult the global configuration file for info (default: {gc.relevance})
  • {string} -- Strategy used for precursor (precursor_prioritization) – prioritization, as a string. There are a limited number of available options - consult the global configuration file for info (default: {gc.heuristic})
  • {number} -- Maximum number of buyable trees to return. Does (max_trees) – not affect expansion time (default: {25})
  • {number} -- Maximum price (max_ppg) – buyable, and thus potentially usable as a leaf node (default: {1e10})
  • {list} -- Reactions to forbid during expansion, (known_bad_reactions) – represented as list of reaction SMILES strings. Each reaction SMILES must be canonicalized, have atom mapping removed, and have its reactant fragments be sorted. Forbidden reactions are checked when processing children returned by the RetroTransformer (default: {[]})
  • {list} -- Molecules to forbid during expansion, (forbidden_molecules) – represented as a list of SMILES strings. Each string must be canonicalized without atom mapping. Forbidden molecules will not be allowed as intermediates or leaf nodes (default: {[]})
  • {number} -- Maximum number of templates to apply at (template_count) – each expansion (default: {100})
  • {string} -- Mode to use for precursor scoring (precursor_score_mode) – when using the SCScore prioritizer and multiple reactant fragments must be scored together (default: {gc.max})
  • {number} -- Maximum cumulative template (max_cum_template_prob) – probability (i.e., relevance score), used as part of the relevance template_prioritizer (default: {1})
  • {dict} -- Dictionary defining a potential chemical (max_natom_dict) – property stopping criterion based on the number of atoms of C, N, O, and H in a molecule. The ‘logic’ keyword of the dict refers to how that maximum number of atom information is combined with the requirement that chemicals be cheaper than max_ppg (default: {defaultdict(lambda: 1e9, {‘logic’: None})})
  • {dict} -- Dictionary defining a potential (min_chemical_history_dict) – chemical stopping criterion based on the number of times a molecule has been seen previously. Always uses logical OR
